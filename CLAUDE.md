# CLAUDE.md

## Project Overview

canvas-mcp is a **Model Context Protocol (MCP) server** that exposes Canvas LMS data to Claude Desktop (or any MCP-compatible client). It wraps the Canvas REST API and provides 15 tools that let an AI assistant query courses, assignments, grades, files, pages, modules, announcements, and more on behalf of a student.

The server communicates over **stdio** using the MCP SDK and is designed to be launched as a subprocess by Claude Desktop.

## Tech Stack

- **Language:** TypeScript (strict mode, ES2022 target)
- **Runtime:** Node.js >= 18
- **Build:** `tsc` (TypeScript compiler) -- outputs CommonJS to `dist/`
- **MCP SDK:** `@modelcontextprotocol/sdk` ^1.0.0
- **PDF parsing:** `pdf-parse` ^2.4.5 (for reading course PDFs)
- **Environment:** `dotenv` ^16.4.5 (loads `.env` at startup)

## Directory Structure

```
canvas-mcp/
  src/
    index.ts            # MCP server setup, tool definitions, tool dispatch (switch/case)
    tools.ts            # All tool implementation functions (business logic)
    canvas-client.ts    # HTTP helpers (fetchAllPages, fetchOne), env validation, Canvas type interfaces
  dist/                 # Compiled JS output (generated by `npm run build`)
  .env                  # CANVAS_TOKEN and CANVAS_DOMAIN (never commit)
  package.json
  tsconfig.json
```

## Key Files

- **`src/index.ts`** -- Entry point. Creates the MCP `Server`, registers `ListToolsRequestSchema` (tool metadata) and `CallToolRequestSchema` (tool dispatch). Validates environment on startup. Connects via `StdioServerTransport`.
- **`src/tools.ts`** -- Contains all 15 tool functions. Each returns a formatted string. Uses helpers like `submissionStatus()`, `isOverdue()`, `daysFromNow()`, and `parsePdfFromUrl()`. All Canvas API calls go through `fetchAllPages`/`fetchOne` from the client module.
- **`src/canvas-client.ts`** -- Builds `BASE_URL` and `AUTH_HEADERS` from env vars. Implements paginated fetching (follows Canvas `Link: rel="next"` headers). Exports all TypeScript interfaces for Canvas API response types.

## Available Tools (15 total)

| Tool | Parameters | Description |
|------|-----------|-------------|
| `get_courses` | none | List active enrolled courses |
| `get_assignments` | `course_id` | Assignments for one course with submission status |
| `get_all_assignments` | none | All assignments across all courses, sorted by due date |
| `get_upcoming_assignments` | `days?` (default 7) | Assignments due within N days |
| `get_calendar_events` | none | Calendar events for next 30 days |
| `get_submission_status` | `course_id` | Per-assignment submission state for a course |
| `get_grades` | `course_id` | Current/final grades + per-assignment scores |
| `get_course_modules` | `course_id` | Modules with item completion status |
| `get_announcements` | none | Recent announcements across all courses |
| `get_todo_items` | none | Canvas to-do list |
| `get_course_files` | `course_id` | List files in a course |
| `get_course_pages` | `course_id` | List wiki/content pages in a course |
| `get_page_content` | `course_id`, `page_url` | Read full text of a specific page |
| `read_course_file` | `course_id`, `file_id` | Download and parse a file (PDF text extraction) |
| `get_module_item_content` | `course_id`, `module_name` | Read content of all items in matching modules |

## Environment Variables

The server requires two environment variables, loaded from `.env` via dotenv:

- `CANVAS_TOKEN` -- Canvas LMS API access token (generate from Canvas Account > Settings > New Access Token)
- `CANVAS_DOMAIN` -- Canvas instance hostname, e.g. `yourschool.instructure.com` (no `https://` prefix)

If either variable is missing, the server throws immediately on startup with a clear error message.

## Architecture Notes

- **Pagination:** `fetchAllPages()` in `canvas-client.ts` follows the Canvas `Link` header `rel="next"` pagination until all pages are consumed. All list endpoints use this.
- **Submission status:** The code explicitly uses `include[]=submission` on assignment fetches to get the *current user's* submission state, rather than relying on `has_submitted_submissions` which is an instructor-level aggregate field.
- **PDF extraction:** `read_course_file` and `get_module_item_content` can download PDF files from Canvas and extract their text content using `pdf-parse`.
- **Error handling:** Tool dispatch catches errors; `McpError` instances are re-thrown (MCP protocol errors), while other errors are returned as `isError: true` responses with the message.
- **HTML stripping:** Several tools strip HTML tags from Canvas content using a simple regex (`/<[^>]*>/g`) and decode common HTML entities.

## Coding Conventions

- TypeScript strict mode enabled
- CommonJS module output (not ESM), but source uses ESM-style imports (handled by `esModuleInterop`)
- Import paths use `.js` extensions (required for Node.js resolution of compiled output)
- Functions return formatted plain-text strings (not JSON) for human-readable tool output
- No linter or formatter configuration is present
- No test framework is configured
